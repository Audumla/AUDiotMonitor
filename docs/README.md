## Documentation Principles

- Start with architecture intent before implementation details.
- Keep component boundaries explicit (`foundation`, `exporter`, `adapters`, `dashboard`, `operations`).
- Provide runnable examples close to the relevant spec area.
- Prefer agent-scoped reading paths under `specifications/components/`.
